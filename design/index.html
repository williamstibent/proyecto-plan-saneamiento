Alcance de cada opción

Las 5 pantallas críticas, en las 3 opciones.

Cada archivo index.html contiene un showcase scrolleable con anclas de navegación a las 5 pantallas. Diseñado para revisar e iterar en equipo.

🔐
01 · Login

Layout asimétrico. Cognito + SSO + MFA opcional. Detección de dispositivo para redirigir al operario en móvil.

📊
02 · Dashboard admin

Sidebar + KPIs (cumplimiento, vencidas, NC), cumplimiento por programa, bandeja de validación, próxima INVIMA.

🧪
03 · Wizard POE

Pasos · editor central · preview vivo del operario. Productos con dosificación, dosis de choque, vínculo de temporizadores.

🗺
04 · Mapa interactivo

Plano SVG con polígonos por área, código de color por estado, operarios activos en tiempo real, panel de detalle.

📱
05 · Móvil PWA

3 pantallas mobile-first 375px: lista de tareas del día, brief de tarea (EPP + productos), checklist + timer de contacto.

Stack de implementación

Listo para React JS.

Los diseños están construidos con Tailwind CDN y clases utility puras, idénticas a las que se usarán en producción con Tailwind v4 + React 19. La traducción a JSX es prácticamente directa.

⚛
React 19 + TypeScript
Componentes funcionales, sin React.FC, hooks + TanStack Query v5.
🎨
Tailwind CSS v4
Las clases del mock son exactamente las de producción. Tokens en CSS vars listos para migrar.
🧩
shadcn/ui + Lucide icons
Componentes base instalables a demanda. Iconografía consistente y accesible.
📱
Vite PWA + Workbox
Service Worker para cachear vista del día. Las fotos se encolan y sincronizan al haber red.
🗺
react-konva
Para el plano interactivo. Polígonos como coordenadas relativas (% del ancho/alto del plano).
Próximos pasos

Cómo seguimos.

  1. 1
    Elegir dirección visual
    Revisa las 3 opciones y selecciona la que mejor representa la marca SanitIA. También se pueden mezclar elementos.
  2. 2
    Validar con 2 usuarios reales
    1 operario + 1 admin-cliente · 15 minutos de tareas guiadas. Observar dónde dudan.
  3. 3
    Tokens y design system
    Extraer colores, radios, sombras y tipografía a CSS vars + Tailwind config. Crear primitives en /shared/components.
  4. 4
    Componentes críticos primero
    TaskCard, ChecklistStep, ContactTimer, ProductDoseCard, AreaPolygon. Tests con Vitest + RTL.
  5. 5
    Iterar con el MVP Fase 1
    Conectar al backend Spring Boot, integrar Cognito, S3 presigned URLs para evidencia.
